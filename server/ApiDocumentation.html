Facilities:

Response:

Response and error response is always 200 in JSON by default.
Response with received data in JSON: {"data":[{"id":"1","name":"Room 1"}],"server":{"status":200,"msg":"OK"}}
Error in JSON: {"server":{"status":405,"code":"1","information":"http://booker/server/ErrorCodeInformation.html"}}
If you want to receive error responses and error code in headers, add this to the end of the query "?response_type_head=true".

Formats:

You can also get the response in different formats.
Add the desired format to the end of the request:
.json(default) - receive response in JSON format.
.xml - receive response in XML format.
.txt - receive text response.
.xhtml - receive HTML page.

Admin:

users:

[GET] booker/client/api/admin/users/hash(admin)/id(user)
Getting full information about the user(s).
/hash(admin) - get all users.
/hash(admin)/id(user) - get the user by ID.

[POST] booker/client/api/admin/users/
Registration - write a new user in table.
hash(admin) | name | password | email - input.

[PUT] booker/client/api/admin/users/
Update - update user data in table.
hash(admin) | id(user) | name | password | email - input.

[DELETE] booker/client/api/admin/users/hash(admin)/id(user)
Delete - removing user (not the administrator) from table.
/hash(admin)/id(user) - input

User:

users:

[PUT] booker/client/api/user/users/
Login - user authorization, if it is in the table, we write a new hash.
name | password - input.
Return user info array whith new hash.

[DELETE] booker/client/api/user/users/hash
Logout - removing (updating) a hash in tables.
/hash - input

events:

[GET] booker/client/api/user/events/id(event)
Receiving event information by id.
/id - input.

[POST] booker/client/api/user/events/
Add new event.
Inputs:
userId | roomId | desc | timeCreate(timestamp) | timeStart(timestamp) | timeEnd(timestamp) | type(Weekly | Bi-weekly | Monthly) | duration(Weekly 1-4 | Bi-weekly 1-2).
Return array whith info.

[PUT] booker/client/api/user/events/
Event Update(s)
hash(user) | userId | roomId | eventId | startHour(8-20) | startMinutes(00 | 30) | endHour(8-20) | endMinutes(00 | 30) | desc | reacurring(TRUE | FALSE)
Return array whith info.

[DELETE] booker/client/api/user/events/hash(user)/id(event)/false(true)
Delete event(s)
/hash(user)/id(event)/false - 1 event
/hash(user)/id(event)/true - reacurring
